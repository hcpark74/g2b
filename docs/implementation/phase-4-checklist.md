# Phase 4 체크리스트

Phase 4 운영 안정화를 실행/검증하기 위한 체크리스트다.

## 1. 운영 실행 기준선

- [ ] 일일/수시 배치 실행 시나리오 정의
- [x] 최근 실행 결과 확인 경로 정리
- [x] 실패 시 재실행/복구 절차 문서화

실행 메모:

- 최근 실행 결과 확인 경로는 `/operations`, `/api/v1/health`, `/admin/sync/*`로 고정한다.
- `/operations`는 health 요약, 최근 실패 건수, 최신 작업 상태, 실패 로그 판독을 함께 제공한다.
- 초기 운영 런북은 `docs/operations-runbook.md`에 정리했다. health 확인, 로그 판독, 수동 재실행 기준, 필수 환경변수를 포함한다.

## 2. 수집 파이프라인 운영화

- [ ] API 동기화와 브라우저 크롤링 분리 여부 결정
- [ ] 배치 단위, 재시도 정책, 중복 방지 규칙 확정
- [ ] 장시간 실행 작업 timeout/abort 기준 정리

## 3. 모니터링/알림

- [ ] 운영 지표 계산 기준 확정
- [ ] 이상 탐지 임계치 확정
- [x] Slack/이메일/웹훅 알림 채널 1차 결정
- [x] 크롤러 health check 정의

실행 메모:

- 1차 알림 채널은 `OPS_SLACK_WEBHOOK_URL` 기반 Slack webhook으로 고정했다. failed sync job 기록 시 공통 로깅 경로에서 자동 전송한다.
- health check는 `/api/v1/health`로 제공한다. `status`, `database`, `recent_failed_jobs`, `latest_job`를 기준으로 `ok/degraded/down` 상태를 판정한다.
- 현재 health 판정은 최근 실패 로그와 DB 접근 가능 여부 중심의 최소 운영 기준선으로 둔다.
