# Frontend Wireframes

## 1. 목적

- 본 문서는 Bootstrap 5 기반으로 주요 화면의 와이어프레임 구조를 정의한다.
- 시각 디자인보다 정보 배치, 사용자 흐름, 컴포넌트 우선순위를 고정하는 데 목적이 있다.

## 2. 공통 레이아웃

```text
+----------------------------------------------------------------------------------+
| Top Header                                                                       |
+----------------------+-----------------------------------------------------------+
| Left Navigation      | Main Content Area                                         |
|                      |                                                           |
| - 입찰 공고          |  Screen-specific content                                  |
| - 사전 탐색          |                                                           |
| - 사후 분석          |                                                           |
| - 관심 공고          |                                                           |
| - 운영 현황          |                                                           |
+----------------------+--------------------------------------------+--------------+
|                      | Main Content Area                           | Right Drawer |
|                      |                                            | (optional)   |
+----------------------+--------------------------------------------+--------------+
```

## 3. 메인 화면 와이어프레임

화면명: `입찰 공고`

### 3.1 전체 구조

```text
+----------------------------------------------------------------------------------+
| Header: 서비스명 | 마지막 동기화 | 빠른 검색 | 사용자 메뉴                         |
+----------------------+-----------------------------------------------------------+
| Navigation           | Page Title: 입찰 공고                    [수동 동기화]    |
|                      +-----------------------------------------------------------+
| - 입찰 공고          | Summary Cards                                            |
| - 사전 탐색          | [신규 공고] [오늘 마감] [관심 공고] [상태 변경 감지]     |
| - 사후 분석          +-----------------------------------------------------------+
| - 관심 공고          | Filter Bar                                               |
| - 운영 현황          | [키워드] [기관] [상태] [업무구분] [마감일] [더보기]       |
|                      +-----------------------------------------------------------+
|                      | Table Toolbar                                             |
|                      | [전체 n건] [정렬] [엑셀 내보내기]                         |
|                      +-----------------------------------------------------------+
|                      | Bid Table                                                 |
|                      | --------------------------------------------------------- |
|                      | |★|상태|구분|공고번호|공고명|기관|추정가|마감|개찰|변경| |
|                      | | |    |    |       |      |    |      |    |    |    | |
|                      | --------------------------------------------------------- |
|                      | Pagination                                                |
+----------------------+--------------------------------------------+--------------+
|                      |                                            | Detail Drawer|
+----------------------+--------------------------------------------+--------------+
```

### 3.2 상단 헤더

구성:

- 서비스명 로고/텍스트
- 마지막 동기화 시각
- 빠른 검색
- 사용자/관리 메뉴

Bootstrap 권장:

- `navbar`
- `input-group`
- `dropdown`

### 3.3 요약 카드 영역

카드 4종 권장:

- 신규 공고
- 오늘 마감
- 관심 공고
- 상태 변경 감지

Bootstrap 권장:

- `row`
- `col`
- `card`
- `badge`

### 3.4 필터 바

1차 노출 필터:

- 키워드
- 공고기관
- 수요기관
- 상태
- 업무구분
- 마감일 범위

확장 필터:

- 업종 제한 여부
- 예산 범위
- 참가가능지역
- 변경 공고 여부

Bootstrap 권장:

- `form-control`
- `form-select`
- `accordion`
- `btn-outline-secondary`

### 3.5 테이블 영역

필수 컬럼:

- 관심 여부
- 상태
- 업무구분
- 공고번호
- 공고명
- 공고기관
- 수요기관
- 추정가격
- 마감일시
- 개찰일시
- 변경 여부

보조 표시:

- 첨부 존재 아이콘
- 마감 임박 강조
- 재공고 여부 배지

Bootstrap 권장:

- `table`
- `table-sm`
- `table-hover`
- `sticky-top` 커스텀 헤더

## 4. 상세 Drawer 와이어프레임

### 4.1 구조

```text
+-----------------------------------------------+
| Drawer Header                                 |
| 공고명                              [닫기]    |
| [상태배지] [즐겨찾기] [다시 읽어오기]         |
+-----------------------------------------------+
| Tab Nav                                       |
| [개요] [자격/제한] [첨부파일] [타임라인] [이력] |
+-----------------------------------------------+
| Tab Content                                   |
|                                               |
| 개요 탭 예시                                  |
| - 공고번호 / 차수                             |
| - 공고기관 / 수요기관                         |
| - 공고일 / 마감일 / 개찰일                    |
| - 계약체결방법 / 추정가격 / VAT               |
| - 외부 링크                                   |
|                                               |
+-----------------------------------------------+
```

Bootstrap 권장:

- `offcanvas offcanvas-end`
- `nav nav-tabs`
- `list-group`
- `btn-group`

### 4.2 탭별 내용

#### `개요`

- 공고 기본 정보
- 예산/추정가/기초금액 요약
- 외부 공고 링크

#### `자격/제한`

- 업종 제한 여부
- 면허 제한 목록
- 허용 업종
- 참가 가능 지역
- 공동수급 방식

#### `첨부파일`

- 일반 첨부
- e발주 첨부
- 혁신장터 최종 제안요청서 첨부

#### `타임라인`

- 사전규격
- 입찰공고
- 개찰/낙찰
- 계약

#### `이력`

- 변경일시
- 변경 항목
- 변경 전/후 값

## 9. 통합 검색/사전 탐색용 확장행 와이어프레임

```text
+----------------------------------------------------------------------------------+
| No | 단계구분 | 업무구분 | 사업명 | 사업번호 | 사업일자 | 기관 | ...          |
+----------------------------------------------------------------------------------+
| 1  | 사전규격 | 용역     | 2026년 ... | R26... | 2026/03/10 | ... | [더보기] |
+----------------------------------------------------------------------------------+
| [진행] [미진행]                                                               |
| [발주계획] -> [요청접수] -> [사전규격] -> [입찰공고] -> [개찰] -> [계약] ... |
| 각 카드에는 번호/일자/없음/상태를 함께 표시                                 |
+----------------------------------------------------------------------------------+
```

활용 목적:

- 여러 건의 사업 진행단계를 빠르게 비교
- 타임라인 전환 여부를 리스트 안에서 바로 확인
- 별도 상세 진입 전 진행 정도를 1차 판단

Bootstrap 권장:

- 확장행 컨테이너: `collapse` 또는 서버 렌더링된 추가 `tr`
- 단계 카드: `card`
- 상태 표기: `badge`
- 연결 표시: 아이콘 또는 단순 화살표

## 11. 결론

- 첫 구현은 `메인 리스트 + 우측 Drawer`에 집중하는 것이 가장 효율적이다.
- Bootstrap 기준으로는 `navbar`, `card`, `table`, `offcanvas`, `tabs`, `badge`, `form`만 잘 조합해도 실무형 UI를 빠르게 만들 수 있다.

## 11. 실제 나라장터 화면 기준 보정 포인트

레퍼런스 화면 기준으로 와이어프레임에 반영할 보정 사항:

- 필터 박스는 테이블 상단의 독립된 회색 패널 형태가 적절하다.
- 상단에 `유의사항` 안내 영역을 접을 수 있는 정보 박스로 둘 수 있다.
- 테이블 헤더는 단색이 아니라 약한 회색 배경 + 얇은 경계선 구조가 실무 친화적이다.
- 결과 목록 제목과 총 건수/엑셀 다운로드/페이지 크기 제어는 테이블 바로 위에 둔다.
- 컬럼 수가 많아질 수 있으므로 좌우 스크롤과 고정 컬럼 전략을 같이 검토해야 한다.
