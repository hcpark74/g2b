# Frontend Implementation Checklist

## 1. 목적

- 본 문서는 프론트엔드 구현 착수 전/중/후에 확인해야 할 항목을 체크리스트 형태로 정리한다.
- 기준 문서는 `docs/frontend/frontend-architecture.md`, `docs/frontend/frontend-wireframes.md`, `docs/frontend/frontend-template-structure.md`, `docs/frontend/frontend-ui-tokens.md`를 따른다.

## 2. 구현 전 체크

### 기술 기준 확정

- [ ] 프론트 스택을 `Bootstrap 5 + Jinja2 + HTMX`로 확정
- [ ] 우측 상세 패턴은 `Drawer 기본`, `확장행 보조`로 확정
- [ ] 메인 운영 허브는 `입찰공고 리스트` 화면으로 확정
- [ ] HTMX 부분 갱신 대상 ID를 확정 (`#bid-table-container`, `#bid-drawer-container`)

### 문서 정합성 확인

- [ ] `docs/PRD.md`의 프론트 스택과 설계 문서가 일치하는지 확인
- [ ] `docs/api/internal-bids.md`와 화면 요구사항이 연결되는지 확인
- [ ] `docs/frontend/frontend-column-priority.md` 기준으로 컬럼 범위를 확정
- [ ] `docs/frontend/frontend-stage-card-spec.md` 기준으로 타임라인 단계 범위를 확정
