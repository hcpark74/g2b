# Development Plan

`docs/PRD.md`를 기준으로 현재 코드베이스 상태를 반영한 개발 실행 계획이다.

## 1. 목표

- 입찰 공고 수집부터 상세 조회, 관심 관리, 상태 추적까지의 핵심 운영 흐름을 MVP로 완성한다.
- 샘플 데이터 중심 화면을 실제 SQLite/SQLModel 기반 조회로 전환한다.
- G2B OpenAPI + Playwright 하이브리드 수집 구조를 단계적으로 안정화한다.
- 운영 전환 전까지 관측 가능성, 실패 복구, 보안 기본선을 확보한다.

## 2. 현재 상태 요약

- FastAPI + Jinja2 + HTMX 기반 화면 구조가 있다.
- `repository -> service -> mapper -> view-model -> template` 계층 분리가 잡혀 있다.
- 샘플 데이터 기반 UI 검증이 가능하다.
- SQLite 적재와 G2B 입찰공고정보서비스 1차 동기화 커맨드가 있다.
- 아직 PRD 기준 전체 범위를 충족하려면 실제 DB 전환, 추가 API 연동, 크롤링, 운영 기능 보강이 필요하다.

## 3. 개발 원칙

- 기능 추가보다 데이터 신뢰성과 추적 가능성을 우선한다.
- UI는 메인 입찰 공고 리스트와 우측 상세 Drawer 흐름을 중심으로 고도화한다.
- 외부 연동은 서비스별로 분리하고, 실패 로그와 재시도 정책을 반드시 남긴다.
- 초기 구현은 SQLite 친화적으로 진행하되 PostgreSQL 전환 가능성을 유지한다.
- 모든 단계는 수동 검증 경로와 자동 테스트를 함께 확보한다.
