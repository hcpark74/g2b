# Phase 3 Checklist

Phase 3 업무형 UI 고도화를 실행/검증하기 위한 체크리스트다.

## 0. 첫 구현 범위

- [ ] 원공고/정정본/취소본/최신 유효본 용어 확정
- [ ] 메인 리스트 버전 표시 규칙 확정
- [ ] 상세 Drawer 버전 이력 표시 규칙 확정
- [ ] timeline/history 정정/취소 이벤트 표현 규칙 확정
- [ ] 정정공고/취소공고 테스트 시나리오 추가
- [x] `bid_version_changes` 모델/테이블 초안 추가
- [x] `sync_bid_change_history` CLI 초안 추가
- [x] phase2 batch/admin batch에 change history sync 포함
- [x] Drawer history/timeline에 change history 항목 연결

## 1. 범위 확정

- [ ] Phase 3 대상 화면 우선순위 확정
- [ ] 메인 리스트/상세 Drawer/결과 화면 역할 분리 확정
- [ ] 수동 액션과 조회 화면 경계 확정
- [ ] 정정공고/취소공고 버전 표시 범위 확정

## 2. 메인 리스트/필터

- [ ] 검색어, 상태, 기관, 기간, 관심 여부 필터 정리
- [ ] 정렬/페이지 크기/총 건수 UX 정리
- [ ] 테이블 컬럼 우선순위와 스크롤 정책 정리
- [ ] 데이터 그리드 도입 여부와 HTMX 연동 경계 정리
- [ ] empty state와 로딩 표현 정리

## 3. 상세 Drawer

- [ ] overview/qualification/attachments/timeline/history 섹션 재정리
- [ ] 크롤링 본문/요약 표시 규칙 정리
- [ ] 계약과정통합공개 결과 강조 방식 정리
- [ ] placeholder fallback 정리
- [ ] 정정/취소/최신본 상태 표시 규칙 정리

## 4. 업무 액션

- [ ] 관심 공고 토글 UX 보강
- [ ] 내부 상태 변경 UX 보강
- [ ] 수동 detail/contract/crawl 재실행 액션 초안 구현
- [ ] 액션 성공/실패 피드백과 `/operations` 연결 확인
- [ ] 크롤링 health/실패 이력 노출 규칙 정리

## 5. 타임라인/결과 고도화

- [ ] timeline stage badge/card 표현 정리
- [ ] history와 timeline 역할 분리 반영
- [ ] `/results` 화면 정보 밀도와 가독성 보강
- [ ] 상태 문구/배지 스타일 통일
- [ ] 정정/취소 이력의 버전 흐름 표시 확인

## 6. 보조 화면

- [ ] 사전 탐색 화면 최소 범위 확정
- [ ] 사후 분석 화면 최소 범위 확정
- [ ] 실제 조회 데이터 연결 확인
- [ ] 측정 가능한 운영 지표와 이상 탐지 후보 정리

## 7. 내보내기/운영성

- [ ] Excel 내보내기 컬럼 범위 확정
- [ ] HTMX partial/query param 계약 정리
- [ ] 운영 로그와 수동 액션 추적 가능성 확인

## 8. 검증

- [ ] `pytest`
- [ ] 메인 리스트 필터/정렬/페이징 테스트
- [ ] 상세 Drawer 섹션 렌더링 테스트
- [ ] 수동 액션 성공/실패 테스트
- [ ] `/results`, `/operations` 회귀 확인
- [ ] 정정공고/취소공고 시나리오 검증
- [ ] 수동 검증 시나리오 정리

## 9. 완료 기준

- [ ] 사용자가 리스트에서 검색, 필터, 상세 확인, 관심 등록, 상태 변경을 한 흐름으로 수행한다.
- [ ] 타임라인과 결과 화면에서 진행 상태를 판독할 수 있다.
- [ ] 수동 액션 결과가 운영 로그에 남는다.
- [ ] 사용자가 원공고, 정정본, 취소본, 최신 유효본을 혼동하지 않는다.
- [ ] 보조 화면과 내보내기가 최소 1개 실무 시나리오를 지원한다.
