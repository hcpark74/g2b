# Frontend Design Guide

## 1. 목적

- 본 문서는 프론트 화면의 시각 원칙과 UI 일관성 규칙을 정의한다.
- 목표는 나라장터에 익숙한 실무자가 낯설지 않게 사용하면서도, 기존보다 빠르게 판단할 수 있는 인터페이스를 만드는 것이다.
- 구현 기반은 `Bootstrap 5`를 전제로 한다.

## 2. 디자인 방향

- 레거시 친화적이되 답답하지 않게
- 데이터 밀도는 높게, 장식은 낮게
- 정보 위계는 명확하게
- 테이블은 중심, Drawer는 보조

## 3. 기본 톤 앤 매너

- 기본 스타일은 `blue-grey + off-white + muted accent`
- 색은 차분하게, 상태는 분명하게
- 시각 효과보다 데이터 판독성과 작업 속도를 우선한다.
- 기본 Bootstrap 스타일을 그대로 쓰기보다, 밀도와 색상만 프로젝트 톤에 맞게 조정한다.

## 4. 주요 시각 규칙

### 테이블

- 헤더는 본문보다 1단계 더 진하게
- 행 간격은 촘촘하되 클릭 영역은 충분히 확보
- hover는 약하게, 선택 상태는 분명하게 표시
- Bootstrap `table`을 기본으로 사용하고, 업무형 밀도에 맞게 셀 패딩을 줄인다.

### 배지

- 상태는 배지로 통일
- 색만으로 의미를 전달하지 말고 텍스트를 함께 제공
- Bootstrap `badge`를 기본 상태 표현으로 사용한다.

### 링크와 액션

- 다운로드, 상세 이동, 외부 링크는 아이콘과 텍스트를 함께 사용
- 위험 액션보다 운영 액션을 우선 배치
- 1차 액션은 Bootstrap `btn` 계열로 통일하고, 우선순위에 따라 `primary`, `secondary`, `outline`을 구분한다.

## 5. 상태 표현 규칙

- `수집완료`: 차분한 회색/청색
- `검토중`: 파란 계열 강조
- `관심`: 별표 + amber 계열
- `투찰완료`: 녹색 계열
- `변경감지`: orange 계열
- `마감임박`: red 계열 보조 강조

## 6. 정보 우선순위 규칙

### 메인 리스트에서 항상 보여야 하는 것

- 공고명
- 공고번호
- 기관명
- 마감일시
- 상태
- 예산 또는 추정가격

### Drawer에서 우선 보여야 하는 것

- 공고 기본 정보
- 참가 자격 핵심
- 첨부파일
- 타임라인

### 접어도 되는 것

- 세부 품목 목록
- 변경 이력 원문
- 원본 API 데이터

## 7. HTMX/서버 렌더링 UX 규칙

- 부분 갱신 시 레이아웃 점프를 줄인다.
- 로딩 상태는 작은 인라인 스켈레톤 또는 텍스트로 표현한다.
- 행 단위 갱신이 가능하면 전체 테이블을 다시 렌더링하지 않는다.

## 8. 결론

- 프론트는 나라장터를 닮되 더 빠르고 명확해야 한다.
- 핵심은 `리스트 중심`, `빠른 상세`, `단계 연결`, `즉각적인 상태 인지`다.

## 9. 실제 화면 레퍼런스 해석

나라장터 실제 입찰공고목록 화면에서 참고할 요소:

- 파란 상단 메뉴와 회색 본문 배경의 안정적인 대비
- 검색 박스와 결과 테이블의 명확한 분리
- 작은 폰트와 촘촘한 행 높이를 활용한 정보 밀도
- 컬럼 정렬/정렬 아이콘이 포함된 업무형 테이블
- 엑셀 다운로드, 페이지 크기 선택, 상세조건 버튼 같은 실무 액션 중심 구성

개선해서 가져갈 요소:

- 원본보다 필터 구조를 단순화한다.
- 공고 행 클릭 시 별도 이동 대신 Drawer 상세 보기를 제공한다.
- 상태와 변경 여부는 텍스트보다 배지로 더 빠르게 인지되게 한다.
- 중요 컬럼은 유지하되 시각적 잡음을 줄인다.
